# private-budget (privatebudget.org)

Privacy-first budgeting PWA: users import bank-statement CSVs and everything is
parsed, stored (Dexie/IndexedDB), and analyzed **entirely on-device**. React +
Vite + Tailwind, static hosting on Cloudflare Pages, AGPL-3.0, public
open-source repo with outside contributors.

## The invariant that overrides everything else

**No user data may ever leave the device, and the app must make no cross-origin
requests.** The CSP is `connect-src 'self'`; the in-app badge counts
cross-origin requests and must stay at 0. Therefore never add: analytics, CDN
assets/fonts, third-party scripts or embeds, telemetry, error reporting, or any
backend/API call. External links (e.g. the Donorbox donate link) open in a new
tab — nothing third-party runs inside the app. If a feature seems to need a
server, it doesn't belong in the core app.

## Commands

- `npm run dev` — dev server; `public/sample-statement.csv` exercises the
  import wizard
- `npm test && npm run build` — run before declaring any change done (build
  includes the typecheck)
- `npm run e2e` — Playwright end-to-end tests (heavier; run when touching
  import/flows)

## Gotchas

- **Money is integer cents everywhere — never floating point.**
- Security headers (`_headers`, CSP, frame-ancestors) are **generated by the
  build** in `vite.config.ts`; edit them there, not in dist/ or the dashboard.
  CSP changes must keep the header and the `<meta>` tag in sync.
- New statement formats plug into the parser layer (`src/import/types.ts`) —
  don't special-case formats elsewhere.
- Tests use `fake-indexeddb`; database logic must go through Dexie so it stays
  testable.
- Public-repo hygiene: AGPL-3.0 applies to contributions; the name/logo/domain
  are **not** licensed (NOTICE); security issues are reported privately per
  SECURITY.md — never document a vulnerability in a public issue/PR.
- README's privacy table makes verifiable claims (offline-capable, zero
  cross-origin requests, reproducible build) — a change that falsifies any of
  them is a bug even if tests pass.
- The "Changes must be made through a pull request" branch protection on
  `main` exists to stop outside-contributor changes from being auto-accepted,
  not to block the owner. Direct pushes to `main` (bypassing the rule) are
  fine when the user is the one asking for them.
